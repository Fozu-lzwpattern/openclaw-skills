# 场景案例：营销活动全链路

## 场景描述

用户需求：「帮我做一个 315 消费者权益日到店餐饮促销活动，在高达上搭建并发布」

## Phase 1: 项目规划

### OKR 拆解

```
Objective: 315 到店餐饮促销活动上线
├── KR1: 活动策划方案确定
│   ├── Epic 1.1: 市场调研
│   │   └── T001: 调研315同类活动案例 → 市场调研员 [无依赖] [25K]
│   └── Epic 1.2: 方案制定
│       ├── T002: 确定活动主题/机制/人群 → 活动策划员 [T001] [30K]
│       └── T003: 制定预算分配方案 → 预算分析员 [T002] [20K]
│
├── KR2: 活动页面搭建完成
│   ├── Epic 2.1: 素材准备
│   │   └── T004: 准备活动头图和文案 → 内容创作员 [T002] [25K]
│   ├── Epic 2.2: 会场搭建
│   │   ├── T005: 创建高达活动+配置组件 → 会场搭建员 [T002,T004] [60K]
│   │   └── T006: 配置供给数据片 → 供给配置员 [T005] [30K]
│   └── Epic 2.3: 质量检查
│       └── T007: 预览检查+问题修复 → QA审核员 [T005,T006] [25K]
│
└── KR3: 活动正式发布
    └── Epic 3.1: 发布上线
        ├── T008: 发布诊断修复 → 发布专员 [T007] [20K]
        └── T009: 正式全量发布 → 发布专员 [T008] [10K]
```

### 团队配置

| 角色 | 人选 | 预算 | label |
|------|------|------|-------|
| 市场调研员 | Sub-agent | 25K | researcher-315 |
| 活动策划员 | Sub-agent | 30K | planner-315 |
| 预算分析员 | Sub-agent | 20K | budget-315 |
| 内容创作员 | Sub-agent | 25K | content-315 |
| 会场搭建员 | Sub-agent | 60K | builder-315 |
| 供给配置员 | Sub-agent | 30K | supply-315 |
| QA审核员 | Sub-agent | 25K | qa-315 |
| 发布专员 | Sub-agent | 30K | publisher-315 |
| CEO 协调 | Main Session | 40K | - |
| **总计** | **8+1** | **285K** | |

### 协作流程

```
调研员 ──→ 策划员 ──→ 预算员
              │
              ├──→ 内容员 ──→ 搭建员 ──→ 供给员 ──→ QA ──→ 发布员
              │               ↑                     ↕
              └───────────────┘                 (迭代修复)
```

阶段划分：
- 阶段1（串行）：调研 → 策划 → 预算
- 阶段2（并行→串行）：内容 & 搭建并行 → 供给 → QA
- 阶段3（迭代）：QA ↔ 搭建 修复
- 阶段4（串行）：发布诊断 → 发布

## Phase 2: 执行示例

### Step 1: Spawn 调研员

```python
sessions_spawn(
    task="""
[OPC 角色卡]
- 角色：市场调研员
- 隶属：315营销项目
- 汇报给：OpenClaw CEO
- 职责：调研315消费者权益日同类活动案例，分析促销机制和效果
- 预算：25K tokens

[Heartbeat 协议]
1. 确认角色和任务
2. 使用 web_search 搜索近两年315促销活动案例
3. 重点搜索：到店餐饮、美团/饿了么/抖音 315 活动
4. 整理为调研报告
5. 汇报产出 + token 消耗

[任务]
调研 315 消费者权益日到店餐饮领域的促销活动案例。
重点关注：活动主题、促销机制（满减/折扣/券）、目标人群、效果数据。

[产出要求]
Markdown 调研报告，包含：
- 至少 3 个案例分析
- 各案例的促销机制对比
- 适合到店餐饮的机制建议
""",
    label="researcher-315"
)
```

### Step 2: 等待调研员 announce → 检查产出 → Spawn 策划员

```python
# 收到调研员产出后
sessions_spawn(
    task="""
[OPC 角色卡]
- 角色：活动策划员
- 隶属：315营销项目
- 职责：基于调研结果制定活动策划方案
- 预算：30K tokens

[上游产出]
{调研员的产出内容}

[任务]
基于上述调研，为 315 到店餐饮促销活动制定策划方案。

[产出要求]
Markdown 策划方案，包含：
- 活动主题和 slogan
- 目标人群画像
- 促销机制设计（主玩法+辅助玩法）
- 时间节点（预热/爆发/长尾）
- 预期效果指标
""",
    label="planner-315"
)
```

### Step 3: 并行阶段 — 同时 Spawn 内容员和搭建员

```python
# 策划方案确认后，同时启动
sessions_spawn(
    task="""
[OPC 角色卡]
- 角色：内容创作员
- 职责：根据策划方案准备活动头图和文案
- 预算：25K tokens

[上游产出 - 策划方案]
{策划员的方案}

[任务]
为活动准备：1. 头图设计方案描述 2. 活动页面文案
""",
    label="content-315"
)

sessions_spawn(
    task="""
[OPC 角色卡]
- 角色：会场搭建员
- 职责：在高达平台创建活动并配置页面组件
- 预算：60K tokens
- 业务 Skill：你可以使用 gundam-ops skill 来操作高达平台

[上游产出 - 策划方案]
{策划员的方案}

[任务]
在高达平台创建 315 到店餐饮促销活动：
1. 创建活动（名称、时间）
2. 添加组件：头图 + 定向展示 + Tab栏 + 商品快筛
3. 基础配置

等内容员产出后 CEO 会发送素材给你配置。
""",
    label="builder-315"
)
```

### Step 4: CEO 协调 — 内容→搭建 的衔接

```python
# 内容员完成后，把素材发给搭建员
sessions_send(
    label="builder-315",
    message="""
内容员已完成素材准备：
- 头图描述：{内容}
- 活动文案：{内容}
请将这些素材配置到对应组件中。
"""
)
```

### Step 5: 搭建完成 → Spawn QA

```python
sessions_spawn(
    task="""
[OPC 角色卡]
- 角色：QA审核员
- 职责：检查活动页面质量
- 预算：25K tokens

[任务]
检查 315 活动页面：
1. 访问预览链接：{preview_url}
2. 检查组件渲染是否正常
3. 检查文案是否正确
4. 检查供给数据是否加载
5. 列出所有问题
""",
    label="qa-315"
)
```

### Step 6: QA ↔ 搭建 迭代修复

```python
# QA 发现问题
sessions_send(
    label="builder-315",
    message="""
QA 发现以下问题，请修复：
1. {问题1}
2. {问题2}
修复后请回复确认。
"""
)
```

### Step 7: 发布

```python
sessions_spawn(
    task="""
[OPC 角色卡]
- 角色：发布专员
- 职责：完成发布诊断和正式发布
- 预算：30K tokens
- 业务 Skill：使用 gundam-ops skill

[任务]
1. 执行发布诊断，修复所有问题
2. 确认预览页面正常
3. 执行全量发布
4. 确认发布成功并获取线上链接
""",
    label="publisher-315"
)
```

## 可裁剪方案

如果项目较简单，可裁剪角色：

### 精简版（3角色）
```
策划员（含调研+预算）→ 搭建员（含内容+供给）→ 发布员（含QA）
预算：~120K tokens
```

### 最简版（1角色）
```
全能执行员（策划+搭建+发布全做）
预算：~80K tokens
```
