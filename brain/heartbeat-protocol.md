# Heartbeat 协议

Sub-agent 执行标准协议，确保行为可控、产出规范。

## 协议流程

```
1. 确认身份
   - 我是谁（角色）
   - 我要做什么（任务）
   - 我的预算是多少

2. 理解上下文
   - 有没有上游产出需要参考
   - 有没有约束条件

3. 执行工作
   - 按职责完成任务
   - 使用允许的工具
   - 在预算内完成

4. 汇报产出
   - 产出内容（按要求的格式）
   - Token 消耗估算
   - 是否完成/部分完成/失败
   - 下游需要知道什么

5. 异常处理
   - 如遇障碍：说明障碍 + 建议方案
   - 如需帮助：明确说明需要 CEO 做什么
   - 如超预算：先汇报已有产出，说明还需多少
```

## 产出汇报模板

```
## 任务完成报告

**角色**：{role}
**任务**：{task}
**状态**：已完成 / 部分完成 / 失败

### 产出
{实际产出内容}

### Token 消耗
约 {N}K tokens

### 备注
{异常情况/建议/下游注意事项}
```

## CEO 检查清单

收到 Sub-agent announce 后：
1. 产出是否符合要求？
2. 是否在预算内？
3. 是否有异常需要处理？
4. 下游任务是否可以触发？
